为什么创建这个工程？

### 1. 背景
* Java因为资源消耗较大，在很多场景下不适用
    * 较慢的启动速度，不适合用来写一些脚本。（例如alfred的脚本）
    * 较大的内存占用，不适合在嵌入式场景下使用。（例如NAS的软件）
    * 较慢的启动速度，不适合Faas场景。
* 所以经常不得不采用各种方式绕过
    * 写脚本时，用其他语言写脚本（例如bash、Python等）。
    * 嵌入式时，用其他的语言，例如PHP。
    * 线上部署时，不能采用FaaS的方式，而是采用传统的部署方式。
* 好消息是终于等来了GraalVM，支持Native Image了。


### 2. 目标
完成Util的集合，降低Java写脚本的成本。
